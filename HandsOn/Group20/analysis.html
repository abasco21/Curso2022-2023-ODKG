Dataset variable analysis

Shared variables by "Asociaciones_202209.csv", "Federaciones_202209.csv", "Colectivos_202209.csv" and "Fundaciones_202209.csv")

"Num. Inscripción"

  • Description: The entity id
  • Data type: Integer [0-9]
  • Comment: Some repeated values, although this is an unique variable (several duplicate instances)
  • Comment(2): No duplicates among different datasets
  • Comment(3): No missing values

"Razón social" ("asociaciones", "federaciones") also named "denominación" (for "colectivos" and "fundaciones")

  • Description: The name of the entity
  • Data type: String [A-Z]
  • Comment: Unique values
  • Comment(2): No missing values

"Tipo_vía", "calle", "número" and "aproximación"

  • Description: Together, they form the complete adress of an entity headquarters
  • Data type: Tipo_vía -> String [A-Z], calle ->String[A-Z], número->[0-9], aproximación->[A-Za-z0-9.-/º]
  • Comment: Discrete set of possible values for "Tipo_via": CALLE, AVENIDA, CAMINO, CAÑADA, BULEVAR, CARRETERA, CUESTA, PASEO, GLORIETA, URB., PLAZA, PASAJE, CARRERA, COLONIA
  • Comment(2): There are missing values
  • Comment(2): Not unique values

"Cod. postal"

  • Description: The ZIP code associated with the adress
  • Data type: Integer [0-9]
  • Comment: Not unique values
  • Comment(2): Some missing values

"Ciudad"

  • Description: The city where adress is located
  • Data type: String [A-Z]
  • Comment: Although our project is focused on Madrid City Area, some entities are placed out of Madrid
  • Comment(2): Solution: don't take in account that instances
  • Comment(3): Not unique values
  • Comment(4): Some mussing values

"Barrio"

  • Description: The subdistrict where adress is located
  • Data type: String [A-Z]
  • Comment: Discrete set of options (one for each of 119 different "barrios" in Madrid
  • Comment(2): Not unique values
  • Comment(3): Some mussing values

"Fecha_registro"

  • Description: The date on which entity was registered
  • Data type: Date
  • Comment: No missing values
  • Comment(2): Not unique values

"Primer_Tlfn"

  • Description: A phone number associated with entity
  • Data type: Integer [0-9]
  • Comment: Some missing values

"Email"

  • Description: An email associated with entity
  • Data type: String [A-Z0-9._%+-]+@[A-Z0-9.-]+\.[A-Z]{2,4}
  • Comment: Some missing values

"Internet"

  • Description: The website of the entity
  • Data type: HTTP URI @^(http\:\/\/|https\:\/\/)?([a-z0-9][a-z0-9\-]*\.)+[a-z0-9][a-z0-9\-]*$@i
  • Comment: Some missing values
  • Comment(2): Some URIs not properly formatted

"Categoría"

  • Description: The type of activity carried out by the entity
  • Data type: String [A-Z]
  • Comment: No missing values
  • Comment(2): For each kind of entity (federación, asociación, colectivo and fundación) there are a predefined discret set of possible values

"Ámbito"

  • Description: The geographical area in which the entity operates
  • Data type: String [A-Z]
  • Comment: No missing values
  • Comment(2): There is a limited set of values (BARRIO, DISTRITO, MUNICIPIO, REGIONAL, ESTATAL, INTERNACIONAL, SIN DETERMINAR)

Variables that are not common to the four datasets

"CIF"

  • Description: The CIF code of the entity
  • Data type: String [A-Z0-9]
  • Belongs to: Asociaciones, federaciones, fundaciones
  • Comment: Some missing values
  • Comment(2): Unique values

"Distrito"

  • Description: The district where adress is located
  • Data type: String [A-Z]
  • Belongs to: Asociaciones
  • Comment: Discrete set of options (one for each of 21 different districts in Madrid
  • Comment(2): Not unique values
  • Comment(3): Some missing values
  • Comment(4): Although District isn't present in 3 of 4 dataset, the variable "Barrio" allows to obtain that information for all instances (all are completely included in a single district)

"Num. socios"

  • Description: The nunber of associates the entity has
  • Data type: Integer [0-9]
  • Belongs to: Fundaciones, asociaciones, colectivos
  • Comment: Some missing values

"Socios totales"

  • Description: The sum of all members belonging to asociations affiliated to a federation
  • Data type: Integer [0-9]
  • Belongs to: Federaciones
  • Comment: No missing values
  • Comment(2): We can't assure that a person belinging to more of one association belinging to a federation won't count double

"Socios Madrid"

  • Description: The sum of all members belonging to asociations affiliated to a federation in the city of Madrid
  • Data type: Integer [0-9]
  • Belongs to: Federaciones
  • Comment: No missing values
  • Comment(2): We can't assure that a person belinging to more of one association belinging to a federation won't count double

"Asoc_totales"

  • Description: The total number of asocitaions belonging to a federation
  • Data type: Integer [0-9]
  • Belongs to: Federaciones
  • Comment: No missing values

"Asoc_Madrid"

  • Description: The total number of asocitaions based in Madrid belonging to a federation
  • Data type: Integer [0-9]
  • Belongs to: Federaciones
  • Comment: No missing values

"NumInscripcion_Fed_Vinculada"

  • Description: The id code (num.inscripción) of the federation to which the entity belongs
  • Data type: Integer [0-9]
  • Belongs to: Asociaciones
  • Comment: Some missing values (there are associations that doesn't belong to any federation)
  • Comment(2): There are duplicates or even triplets for some unique variables because an association can belong to more than one federation simultaneously

"RazonSocial_Fed_Vinculada"

  • Description: The name (razón social) of the federation to which the entity belongs
  • Data type: Integer [0-9]
  • Belongs to: Asociaciones
  • Comment: Some missing values (there are associations that doesn't belong to any federation)
  • Comment(2): There are duplicates or even triplets for some unique variables because an association can belong to more than one federation simultaneously

"Fecha IPM"

  • Description: Unknown
  • Data type: Date
  • Belongs to: Fundaciones
  • Comment: Some missing values
